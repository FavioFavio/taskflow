# Sprint 1 - Authentication

## Objective

Implement a complete authentication system using Supabase SSR.

The feature must be production-ready.

Only implement authentication.

Do NOT start the dashboard.

---

# Scope

Implement

- Login page

- Register page

- Logout

- Session persistence

- Protected routes

- Middleware

- Auth layout

---

# Technical Requirements

Use

- Next.js App Router

- Server Actions

- Supabase SSR

- React Hook Form

- Zod

- shadcn/ui

No mock data.

No REST API.

No Redux.

No Context for auth.

---

# UI Requirements

Modern SaaS design.

Responsive.

Dark mode.

Accessible.

Validation messages.

Loading buttons.

Password hidden by default.
